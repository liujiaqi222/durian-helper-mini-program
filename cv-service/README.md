# Python YOLO 榴莲识图微服务

这个目录放的是“榴莲检测与编号”的 Python 微服务。它只负责一件事：

1. 接收一张图片
2. 用 YOLO 找出图片里的榴莲
3. 返回每个榴莲的边界框坐标和置信度

后续的“按从左到右、从上到下编号”“生成标注图”“调用大模型评分”，建议放在主后端或者单独的编排层处理。

## 1. 这个微服务解决什么问题

下面的命令示例会先约定一个目录别名，减少反复输入 `cv-service`：

```bash
CV_DIR=cv-service
```

对你这个项目来说，YOLO 微服务负责的是第一段：

- 图一 -> 识别出所有榴莲
- 给后面的编号逻辑提供检测框

它**不负责**：

- 推荐哪个榴莲最好
- 输出购买建议文案
- 生成分享图

这样拆分的原因很简单：YOLO 擅长“在哪里”，大模型擅长“怎么看”。

## 2. 适合 CV 小白的理解方式

你可以把 YOLO 理解成一个“看图找目标的模型”。

输入：
- 一张图片

输出：
- 这张图里有哪些榴莲
- 每个榴莲大概在哪个矩形区域里
- 模型对这个判断有多大把握

例如输出结果会像这样：

```json
[
  {
    "class_name": "durian",
    "confidence": 0.94,
    "bbox": {
      "x1": 120,
      "y1": 80,
      "x2": 360,
      "y2": 420
    }
  }
]
```

其中：
- `x1, y1` 是左上角
- `x2, y2` 是右下角
- `confidence` 是置信度，越接近 `1` 越有把握

## 3. 推荐落地路径

建议按下面顺序做，不要一开始就训练复杂模型：

1. 先把微服务跑起来
2. 先用 YOLO 的现成框架接通推理流程
3. 再准备榴莲数据集
4. 再训练你自己的榴莲检测模型
5. 最后把训练好的 `best.pt` 放进来替换默认权重

## 5. 依赖安装

建议先使用 Python 3.11+。

> 说明：`venv` 会“继承你用来创建它的那个 Python 解释器版本”。
> 如果你用 `python3 -m venv .venv` 创建虚拟环境，而你机器上的 `python3` 实际是 3.9.x，
> 那这个服务就会跑在 3.9.x 上，进而出现某些依赖（例如较新的 NumPy）无法安装的问题。

安装依赖：

```bash
CV_DIR=cv-service
cd "$CV_DIR"
python3 --version  # 确认是 3.11+
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
pip install -r requirements.txt
```

## 6. 如何启动

```bash
CV_DIR=cv-service
cd "$CV_DIR"
uvicorn app.main:app --reload --port 8010
```

启动后可以访问：

- `GET http://127.0.0.1:8010/health`

## 7. 当前代码默认假设

当前代码里默认配置的是：

- 模型文件路径：`models/durian-best.pt`
- 类别名：`durian`

注意：这个权重文件现在还没有创建。你后面训练好模型后，把文件放到：

```text
cv-service/models/durian-best.pt
```

即可。

如果暂时没有自训练模型，也可以先换成别的 YOLO 权重做流程联调，但真正上线前，还是建议训练“榴莲”专用模型。

## 8. 训练模型你只需要理解三件事

### 第一步：准备数据

你需要收集很多张榴莲图片，并给每个榴莲画框标注。

训练检测模型的数据，本质上是：

- 图片
- 每张图片里榴莲的框坐标

### 第二步：训练

用 Ultralytics YOLO 训练，产出一个 `best.pt`。

### 第三步：部署

把 `best.pt` 放进 `models/` 目录，微服务启动时加载这个模型。

## 9. 为什么接口先只做检测，不直接做编号

因为“检测”和“编号”是两种不同责任：

- 检测：模型负责看见有哪些榴莲
- 编号：业务规则负责把检测框按固定顺序排列成 A、B、C

把编号放在主后端更好维护，原因是：

- 编号规则可能会改
- 有时需要和前端展示规则保持一致
- 你后面可能会加入“按货架层级排序”等业务逻辑

## 10. 后续建议

这个微服务接好后，主后端可以这样用它：

1. 上传原图
2. 调用 Python 微服务 `/detect`
3. 拿到检测框
4. 主后端负责排序编号
5. 主后端生成标注图
6. 主后端把标注图交给多模态模型评分

## 11. 当前代码是 MVP 骨架

这份代码是“工程骨架”，目标是让你先把服务跑起来，后面再逐步补：

- 自训练权重
- 标注图生成
- 批量检测
- URL 下载图片
- 鉴权
- 监控与日志

如果你下一步要继续推进，最自然的顺序是：

1. 先看新手训练文档：YOLO-TRAINING-BEGINNER.md
2. 再开始准备数据集和标注
3. 训练出 `best.pt` 后放进 `models/`

后续如果继续做工程化，再补：

1. “编号 + 标注图生成”
2. “训练脚本 + 推理脚本”
