# YOLO 榴莲训练新手文档

这份文档是写给“会写代码，但几乎没做过 CV 训练”的人看的。

目标只有一个：

- 训练出一个能识别图片里榴莲位置的 YOLO 模型
- 最终产出 `best.pt`
- 把它接回当前 Python 微服务

如果你是第一次做模型训练，不要一开始追求“高精度”和“论文级效果”。先把完整流程跑通，再慢慢优化。

## 1. 先理解你现在到底要训练什么

你现在训练的是“目标检测模型”，不是“分类模型”。

两者区别：

- 分类模型：判断一张图是不是榴莲
- 目标检测模型：找出图里每一个榴莲在哪里

你的项目需要的是第二种，因为一张图里通常有很多个榴莲，后端后续还要：

1. 根据检测框做排序编号
2. 把每个榴莲单独裁剪出来
3. 再交给大模型做评分和解释

所以这一步的训练目标非常明确：

- 输入一张货架图
- 输出多个榴莲框

## 2. 训练前你需要准备什么

先准备 4 样东西：

1. 一台能跑 Python 的电脑
2. 已经安装好的项目依赖
3. 一批榴莲图片
4. 一个标注工具

你现在已经完成了第 2 步的一部分，因为 `requirements.txt` 已经安装好了。

接下来真正重要的是“图片”和“标注”。

## 3. 数据量要准备多少

新手不要被“上千张起步”吓住。对你这个 MVP 项目，建议从下面这个量级开始：

- 最低可启动：100 到 200 张图片
- 更稳妥：300 到 800 张图片
- 想继续优化：1000 张以上

图片质量比数量更重要。你要尽量覆盖下面这些情况：

- 多个榴莲同框
- 榴莲互相遮挡
- 大果、小果
- 光线偏黄、偏绿、偏暗
- 背景复杂
- 不同拍摄距离
- 角度略有变化

注意：

- 不要只收“拍得很漂亮”的样例图
- 训练集越接近真实用户拍照场景，后面效果越有意义

## 4. 图片从哪里来

数据来源通常有三种：

1. 自己拍
2. 店里实拍
3. 业务允许范围内的历史样例

优先级建议：

1. 真实业务图片
2. 类似业务场景图片
3. 网上公开图片

原因很简单：

- 模型最怕“训练时看的是 A 场景，上线时遇到 B 场景”

## 5. 标注是什么

标注就是“你告诉模型，榴莲在哪里”。

具体做法：

- 打开一张图片
- 用鼠标给每个榴莲画一个矩形框
- 每个框都标成同一个类别：`durian`

这一步是训练里最重要的人工工作。

模型最后学得好不好，跟标注质量关系非常大。

## 6. 推荐你怎么标注

新手最简单的方案是用图形化标注工具，例如 `labelImg`、`CVAT`、`Roboflow` 一类。

你只需要遵守下面几条规则，不需要先研究复杂理论。

### 6.1 类别只有一个

你现在只需要一个类别：

```text
durian
```

不要一上来拆成：

- durian_good
- durian_bad
- durian_open
- durian_closed

这些属于后续更复杂的任务。第一阶段只做“把榴莲找出来”。

### 6.2 框尽量贴边，但不要过度抠细节

目标是把整个榴莲主体框进去。

经验规则：

- 框太小：会截掉榴莲边缘
- 框太大：会把大量背景带进去

你只要做到“主体完整、边缘大致贴合”就够了。

### 6.3 遮挡的榴莲也要标

如果一个榴莲只露出一部分，但你依然能明显判断它是一个独立榴莲，建议也标出来。

原因：

- 真实场景里遮挡很常见
- 不标遮挡目标，模型上线后会更容易漏检

### 6.4 完全看不清的就别硬标

如果只露出一个尖角，连你自己都不确定是不是一个完整榴莲，就不要硬标。

原则：

- 明显独立目标：标
- 高度不确定：不标

### 6.5 同一张图的标注标准要一致

最怕的不是“偶尔有一点不准”，最怕的是前后标准不一致。

例如：

- 有时框很紧
- 有时框特别松
- 同样的遮挡目标，有时标有时不标

这种不一致会让模型学得很乱。

## 7. YOLO 需要什么样的数据目录

Ultralytics YOLO 常用的数据目录长这样：

```text
datasets/durian/
├── images
│   ├── train
│   └── val
├── labels
│   ├── train
│   └── val
└── data.yaml
```

其中：

- `images/train` 放训练图片
- `images/val` 放验证图片
- `labels/train` 放训练标注
- `labels/val` 放验证标注

一张图片会对应一个同名的标注文件。

例如：

```text
images/train/img001.jpg
labels/train/img001.txt
```

## 8. YOLO 标注文件长什么样

YOLO 的单个标注文件通常是纯文本，每一行代表一个目标：

```text
0 0.512 0.487 0.301 0.442
0 0.242 0.530 0.220 0.390
```

你现在不用手写这个文件。标注工具一般会自动导出。

你只需要知道：

- 第一个数字 `0` 是类别 ID
- 因为你只有一个类别 `durian`，所以它永远是 `0`
- 后面四个数字是框的位置和大小，做过归一化

## 9. 训练集和验证集怎么分

建议：

- 训练集：80%
- 验证集：20%

例如你有 200 张图：

- 160 张放 `train`
- 40 张放 `val`

注意：

- 同一个场景连拍的图片，不要一部分进训练集、一部分进验证集太多
- 否则验证结果会“虚高”

原因是模型可能只是记住了非常相似的背景。

## 10. data.yaml 怎么写

在数据集目录下创建 `data.yaml`：

```yaml
path: /绝对路径/datasets/durian
train: images/train
val: images/val

names:
  0: durian
```

说明：

- `path` 最简单的写法就是绝对路径
- `names` 里定义类别
- 这里只有一个类别，所以 `0: durian`

## 11. 最小训练命令

假设你已经进入虚拟环境，并且数据集已经准备好，可以用最简单的方式先跑起来：

```bash
cd /Users/liujiaqi/code/JS/durian-helper-mini-program/server/python-cv-service
source .venv/bin/activate
yolo detect train data=/你的数据集路径/data.yaml model=yolov8n.pt epochs=50 imgsz=640
```

你只需要先理解这几个参数：

- `data`
  - 数据集配置文件
- `model`
  - 训练起点模型
  - `yolov8n.pt` 是比较轻量的起点，适合先跑通
- `epochs`
  - 训练轮数
  - 可以先用 `50`
- `imgsz`
  - 输入图片尺寸
  - 先用 `640` 即可

## 12. 新手推荐训练参数

第一版建议用这个：

```bash
yolo detect train \
  data=/你的数据集路径/data.yaml \
  model=yolov8n.pt \
  epochs=80 \
  imgsz=640 \
  batch=8 \
  device=cpu
```

说明：

- 如果你没有 NVIDIA GPU，先用 `device=cpu`
- CPU 训练会慢，但能跑
- 有 GPU 的话，把 `device=cpu` 去掉，或者改成 `device=0`

如果电脑性能一般，可以把：

- `epochs` 先降到 `30`
- `batch` 先降到 `4`

先确认整条链路没问题，再慢慢调大。

## 13. 训练输出会出现在哪里

Ultralytics 一般会把结果输出到类似目录：

```text
runs/detect/train/
```

里面你重点关注两个东西：

1. `weights/best.pt`
2. 一些训练图和指标图

其中最重要的是：

```text
runs/detect/train/weights/best.pt
```

这就是你后面要接入微服务的模型文件。

## 14. 怎么判断训练有没有成功

新手先不要过度研究几十个指标，先用这几个最直观标准：

1. 模型能不能在测试图上把大部分榴莲框出来
2. 漏检是否明显减少
3. 误检是否可接受
4. 编号前的检测结果是否已经足够业务使用

对你这个项目来说，“业务可用”比“学术指标特别漂亮”更重要。

如果一张图里 9 个榴莲能检出 7 到 9 个，而且框大致合理，MVP 就已经能推进。

## 15. 训练后怎么做一次预测验证

你可以直接用训练好的模型跑几张图看看：

```bash
yolo detect predict \
  model=runs/detect/train/weights/best.pt \
  source=/你的测试图片目录 \
  conf=0.35
```

运行后会在 `runs/detect/predict/` 下生成带框的结果图。

你重点看三件事：

1. 有没有明显漏掉榴莲
2. 有没有把别的水果误认为榴莲
3. 框的位置是否离谱

## 16. 训练好后怎么接入当前微服务

训练完成后，把 `best.pt` 复制到下面这个位置：

```text
server/python-cv-service/models/durian-best.pt
```

也就是最终变成：

```text
server/python-cv-service/
├── models
│   └── durian-best.pt
```

当前微服务默认就会从这个路径加载模型。

## 17. 你实际要执行的最短路径

如果你只想先做出第一个可跑版本，按这个顺序来：

1. 收集 150 到 300 张榴莲图
2. 只标一个类别：`durian`
3. 划分 `train/val`
4. 写好 `data.yaml`
5. 运行训练命令
6. 用 `predict` 看效果
7. 把 `best.pt` 复制到 `models/durian-best.pt`
8. 启动微服务，联调 `/detect`

## 18. 新手最常见的坑

### 18.1 类别名不统一

例如：

- 有的地方叫 `durian`
- 有的地方叫 `Durian`
- 有的地方叫 `榴莲`

建议统一只用：

```text
durian
```

### 18.2 图片有，标签没对上

例如：

- `images/train/img001.jpg`
- 但没有 `labels/train/img001.txt`

或者文件名大小写不一致。

这种会直接影响训练。

### 18.3 标注很随意

比如：

- 有的框极大
- 有的框极小
- 同样的遮挡目标，有时标有时不标

模型通常会因此变得不稳定。

### 18.4 一开始就追求太多类别

你当前阶段不要同时训练：

- 榴莲
- 裂口榴莲
- 优质榴莲
- 劣质榴莲

先把“榴莲检测”单独做好。

### 18.5 训练集和验证集太像

例如一组连拍图拆散后同时进训练和验证，容易让你误以为模型很好。

### 18.6 误以为检测模型能直接输出“哪个好吃”

YOLO 只负责找目标位置，不负责生成购买建议。

你这个项目的推荐逻辑应该是：

1. YOLO 找出榴莲
2. 主后端编号与裁剪
3. 大模型做评分和解释

## 19. 如果你暂时不会自己标注，最现实的做法

可以先这样推进：

1. 先收集一批图
2. 找一个图形化工具手动标注 50 到 100 张
3. 先训练一个很粗糙的模型
4. 用这个粗糙模型去辅助你继续筛图和补标
5. 再训练第二版

这比一开始就想“直接做出很准的模型”更现实。

## 20. 你当前最值得做的下一步

如果你问“我明天就要开始干，第一步是什么”，答案是：

1. 新建一个数据集目录
2. 先收集 100 到 200 张真实榴莲场景图
3. 用一个标注工具把每个榴莲都框出来
4. 导出成 YOLO 格式
5. 按本文命令跑第一版训练

做到这里，你就已经不是“完全不会训练模型”了。
